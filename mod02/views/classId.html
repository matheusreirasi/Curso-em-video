Antes de começar deixarei logo destacado que na utilização de CSS, é possível simplificar a sintaxe do código simplesmente colocando hashtag(#) antes do nome do id criado no HTML e ponto (.) antes do nome da class criada no HTML.

Além disso, a utilização dessas duas tags no HTML faz com que as CSS possuam também dois tipos de seletores, os genéricos e os personalizados.
Os genéricos são todos aqueles que não possuem nenhum atributo passado dentro de sua tag, ou seja, não possuem nem <id> nem <class>.
Em contrapartida os seletores personalizados possuem sua própria marcação (nome da Id ou o nome de uma Class) passada dentro da tag que se queira ser personalizada.

Usando class

O uso de class no HTML serve para especificar no arquivo em CSS o que será editado, serve para diferenciar tags que foram usadas mais de uma vez.
Por exemplo, foi usada a tag <h1> mais de uma vez, porém deseja-se usar diferentes estilos para ambos, então é necessário passar uma class dentro de cada tag especificando o nome daquela tag que será editada no CSS.

É possível também usar class em diferentes tags e tudo que for editado no CSS utilizando essa class será aplicada em todas as tags do HTML que contiverem aquela class.
Também é possível que uma tag HTML possua mais de uma classe, ficando cada nome da classe separado por espaço, por isso deve-se usar class com nomes unitários, ou seja, cada palavra separada por underline, trasso ou começando a próxima palavra com letra maiúscula.
Quando utilizar mais de uma classe na tag, e esse classe tiver algumas formatações iguais no arquivo CSS, então a prioridade será dada para a primeira classe passada dentro da tag e assim por diante. Se a primeira classe passada tiver <font-size:3em> e a segunda classe tiver <font-size:1em>, então a página terá uma fonte com tamanho de 3em.

Pseudo-classes

Uma pseudo-classe CSS é uma palavra adicionada às declarações de um seletor após um sinais de dois pontos (:) e especificam um estado especial de um elemento.
No site da Mozilla existem diversos exemplos.

No caso dessa página eu criei dois seletores code no CSS, ambos genéricos, porém um deles contém uma pseudo-classe fazendo o código que aparece ná pagina ficar azul.
Eu havia tendado colocar ambos no mesmo seletor, sem passar nenhum id ou class, utilizando <code:hover{}>, porém a formatação somente acontecia quando eu passava o ponteiro em cima da palavra. No meu caso não é isso que eu esperava acontecer pois a parte contendo o código deve sempre ficar sempre com o mesmo tamanho de fonte.

Pseudo-elemento

Já um pseudo-elemento funciona de forma parecida com uma pseudo-classe, porém o pseudo-elemento é usado para formatação de um determinado espaço do elemento referenciado.
Outra diferença entre eles é seu código, onde este é acompanhado de dois dois pontos (::) e as pseudo-classes possuem somente um dois pontos (:).

Usando Id

O id serve para identificar um elemento único dentro da página. Essa identificação permite que criemos um estilo especial para esse elemento único.

No HTML devemos usar somente um id para um único elemento, ou seja, não se deve colocar o mesmo id para dois elementos diferentes do site, pois desse jeito, esse elemento não é mais considerado único.

No caso dessa página como exemplo, foi criado no CSS um seletor genérico colocando todo h1 centralizado no meio, um seletor id relacionado ao h1 não especificando a posição desse h1 e um outro seletor class também do h1 colocando o texto alinhado à esquerda. Com isso percebe-se que existe uma hierarquia dos seletores, onde primeiro será aplicado todos os estilos CSS passados dentro dos seletores personalizados (id e class) e depois será aplicado os estilos dos seletores genéricos (quando puder ser complementar àquele seletor personalizado).
